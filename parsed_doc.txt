2. System Overview
The system is split into two clearly separated layers: a Python FastAPI backend that handles AI logic and PDF parsing, and a plain HTML/CSS/JavaScript frontend that handles the user interface. They communicate over HTTP using a REST API.
High-Level Architecture
The frontend and backend can run on the same machine. For the assessment demo, run FastAPI locally with 'uvicorn main:app' and open localhost:8000 in the browser.

6. Frontend — index.html
The frontend is a single HTML file served directly by FastAPI. It uses Tailwind CSS (via CDN) for styling and vanilla JavaScript for all interactivity. No React, no npm, no build step needed.
6.1 What the Frontend Does
Provides a text area for the Job Description input
Provides a multi-file PDF upload zone (drag-and-drop or click)
Loops through each uploaded PDF and sends it to POST /screen-resume
Shows a live progress indicator (Processing 2 of 5...)
Collects all results, sorts by score, and renders a ranked table
Color-codes each row: green = Strong Fit, amber = Moderate, red = Not Fit
Provides a one-click CSV export of the results table

7.3 How the Prompt Ensures Consistent JSON Output
The most common failure mode with AI APIs is inconsistent output format. The prompt is designed with three specific techniques to prevent this:
Explicit instruction: 'Respond ONLY with a valid JSON object, no extra text' — prevents Gemini from adding explanatory prose
Schema definition inline: the exact JSON keys and types are shown in the prompt — Gemini follows the template reliably
Low temperature (0.2): reduces randomness, making the output format stable across multiple runs
JSON fence stripping in code: the call_gemini() function strips markdown code fences in case Gemini wraps output in triple backticks
